#+OPTIONS: toc:nil
* org-grimoire — An Emacs-native Static Site Generator
#+ATTR_HTML: :width 128px
[[file:docs/logo.png]]

An Emacs-native static site generator. Write your content in org-mode,
configure everything in your Emacs config, build with a single command.

No external dependencies.
Just Emacs, org-mode, and HTML templates.

** Installation

Clone the repository and add it to your load path:

#+begin_src elisp
(add-to-list 'load-path "~/path/to/org-grimoire/")
(require 'org-grimoire)
#+end_src

** Configuration

Add to your Emacs config:

#+begin_src elisp
(grimoire-setup "my-blog"
  :base-dir    "~/blog"
  :base-url    "https://yoursite.com"
  :title       "My Blog"
  :description "A blog about things"
  :theme       "mytheme"
  :per-page    10)
#+end_src

org-grimoire derives all paths from =:base-dir= automatically:

| Key          | Derived path                  |
|--------------+-------------------------------|
| =:source=    | =base-dir/content/=           |
| =:output=    | =base-dir/public_html/=       |
| =:static=    | =base-dir/static/=            |
| =:theme=     | =base-dir/themes/mytheme/=    |

You can override any of these individually by passing =:source=,
=:output=, =:static=, or a full path to =:theme= explicitly.

*** Multiple sites

You can configure and build multiple sites independently:

#+begin_src elisp
(grimoire-setup "blog"
  :base-dir "~/blog"
  :base-url "https://blog.com"
  :title    "My Blog"
  :theme    "minimal")

(grimoire-setup "company"
  :base-dir "~/company-site"
  :base-url "https://company.com"
  :title    "My Company"
  :theme    "corporate")

(grimoire-build "blog")
(grimoire-build "company")
#+end_src

** Writing content

Each org file declares its own type via =#+TYPE:=:

#+begin_src org
,#+TITLE: My Post
,#+DATE: 2024-01-15
,#+TYPE: post
,#+TAGS: emacs lisp

Post content goes here. Full org-mode syntax supported.
#+end_src

*** Frontmatter keywords

| Keyword      | Description                           | Example                  |
|--------------+---------------------------------------+--------------------------|
| =#+TITLE=    | Page title                            | =#+TITLE: My Post=       |
| =#+DATE=     | Publication date (yyyy-mm-dd)         | =#+DATE: 2024-01-15=     |
| =#+TYPE=     | Content type, maps to a template      | =#+TYPE: post=           |
| =#+TAGS=     | Space or comma separated list of tags | =#+TAGS: emacs lisp=     |
| =#+DRAFT=    | Set to =t= to exclude from build      | =#+DRAFT: t=             |
| =#+LISTED=   | Set to =nil= to hide from index       | =#+LISTED: nil=          |

*** Draft vs Listed

- =#+DRAFT: t= — post is not built at all, completely excluded
- =#+LISTED: nil= — post is built and accessible via its URL and tag
  pages, but does not appear on the main index listing. Defaults to
  =t= if omitted.

*** Images and file links

org-grimoire automatically detects =file:= links in your org files
and copies referenced files to the output directory, preserving
relative paths. This means org-download works out of the box:

#+begin_src org
[[file:images/screenshot.png]]
#+end_src

Gets copied from =content/posts/images/screenshot.png= to
=public_html/posts/images/screenshot.png= and the relative link works
in the browser with no extra configuration.

** Templates

*** Base template

=base.html= is the site-wide layout. All pages are wrapped in it.
It receives ={{title}}= and ={{content}}= placeholders:

#+begin_src html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>{{title}}</title>
  <link rel="stylesheet" href="/static/css/style.css">
</head>
<body>
  {{include partials/navbar.html}}
  {{content}}
  {{include partials/footer.html}}
</body>
</html>
#+end_src

*** Type templates

Each content type maps to its own template file. =#+TYPE: post= uses
=post.html=, =#+TYPE: page= uses =page.html= and so on.

=post.html=:
#+begin_src html
<article>
  <h1>{{title}}</h1>
  <time>{{date}}</time>
  <div class="tags">{{tags}}</div>
  <div class="content">{{content}}</div>
</article>
#+end_src

Available placeholders in type templates:

| Placeholder    | Description                     |
|----------------+---------------------------------|
| ={{title}}=    | Post title                      |
| ={{content}}=  | Rendered HTML content           |
| ={{date}}=     | Publication date                |
| ={{tags}}=     | Comma separated list of tags    |
| ={{slug}}=     | URL slug derived from filename  |

*** Index template

Used for the post listing and tag pages. =index.html=:

#+begin_src html
<h1>{{title}}</h1>
<ul>{{posts}}</ul>
{{pagination}}
#+end_src

| Placeholder      | Description                      |
|------------------+----------------------------------|
| ={{title}}=      | Page title                       |
| ={{posts}}=      | Generated list of post links     |
| ={{pagination}}= | Previous/next page navigation    |

*** Tags template

=tags.html= is used for the tags index page at =/tags/index.html=:

#+begin_src html
<h1>{{title}}</h1>
<ul>{{tags}}</ul>
#+end_src

*** Includes

Any template can include another file using ={{include}}=:

#+begin_src html
{{include partials/navbar.html}}
{{include partials/footer.html}}
#+end_src

Files are resolved relative to your theme directory.

** Building

Run =M-x grimoire-build= and enter the site name when prompted, or
call it directly from your config:

#+begin_src elisp
(grimoire-build "my-blog")
#+end_src

Output goes to =base-dir/public_html/=. Build log is written to the
=*Messages*= buffer. Errors in individual files are logged as warnings
and the build continues.

** Feeds

RSS and Atom feeds are automatically generated at =/rss.xml= and
=/atom.xml= from all published posts.

** Non-goals

- No markdown support, org-mode only
- No JavaScript build pipeline
- No external dependencies outside of Emacs packages
- No built-in development server (use Python's =http.server= or similar)
